# Unimplemented Webhook Fields

These WhatsApp Cloud API webhook fields are not yet handled by this package.
They are silently dropped (200 returned) so retries are not triggered. Each
section below has enough context to implement support.

Reference: https://developers.facebook.com/docs/whatsapp/cloud-api/webhooks

---

## 1. automatic_events

**What it is:** Notifies you when WhatsApp detects a purchase or lead event in a
chat thread between you and a WhatsApp user who messaged you via your
Click-to-WhatsApp ad. Requires opt-in to Automatic Events reporting.

**Typical payload shape:**
```json
{
  "field": "automatic_events",
  "value": {
    "messaging_product": "whatsapp",
    "metadata": { "display_phone_number": "...", "phone_number_id": "..." },
    "automatic_events": [
      {
        "id": "wamid.xxxx",
        "timestamp": "1749416383",
        "type": "purchase",          // or "lead"
        "from": "16505551234",
        "purchase": {
          "currency": "USD",
          "value": 19.99,
          "catalog_id": "..."
        },
        "lead": {
          "lead_form_id": "...",
          "field_values": [...]
        }
      }
    ]
  }
}
```

**Implementation steps:**
1. Add `ChangeFieldAutomaticEvents ChangeField = "automatic_events"` to `handler.go`
2. Define types in a new `webhooks/automatic_events.go`:
   - `AutomaticEventContext` — wraps notification metadata + entry info
   - `AutomaticEvent` — the event payload (id, timestamp, type, from, purchase/lead)
   - `PurchaseEvent`, `LeadEvent` — type-specific payloads
3. Add to `Handler` struct:
   - `automaticEvent EventHandler[AutomaticEventContext, AutomaticEvent]`
4. Add `OnAutomaticEvent` / `SetAutomaticEventHandler` methods
5. Add `case ChangeFieldAutomaticEvents.String():` in `handleNotificationChange`

## 8. smb_message_echoes

**What it is:** Echoes messages sent via the WhatsApp Business app or a
companion ("linked") device by a business customer onboarded to the
Cloud API. When a business replies from their phone, the Cloud API receives
a copy so all conversations stay in sync.

**Typical payload shape:**
```json
{
  "field": "smb_message_echoes",
  "value": {
    "messaging_product": "whatsapp",
    "metadata": { "display_phone_number": "...", "phone_number_id": "..." },
    "messages": [
      {
        "id": "wamid.xxxx",
        "from": "15550783881",
        "timestamp": "1749416383",
        "type": "text",
        "text": { "body": "Reply from phone" }
      }
    ]
  }
}
```

**Note:** The payload structure is identical to the `messages` webhook —
the difference is the `field` name and that messages come from the
business rather than the customer. You can likely reuse the `Message`
struct and `handleNotificationMessageItem` logic.

**Implementation steps:**
1. Add `ChangeFieldSMBMessageEchoes ChangeField = "smb_message_echoes"`
2. Add to `Handler` struct:
   - `smbMessageEcho MessageHandler[Message]` — reuse existing `Message` type
3. Add `OnSMBMessageEcho` / `SetSMBMessageEchoHandler`
4. Add `case ChangeFieldSMBMessageEchoes.String():` — delegate to `handleNotificationMessageItem`
   since the payload shape matches the `messages` field
